# Design Pattern

> Eine Lösung eines häufig auftretenden Gestaltungsproblems in einem gegebenen
> Nutzungskontext. Das Design Pattern beschreibt ein Gestaltungsproblem,
> eine allgemeine Lösung und Beispiele, wie diese Lösung angewandt
> werden kann.

Ein Design Pattern ist eine Lösung für häufig auftretende Gestaltungsprobleme
in einem bestimmten Nutzungskontext.

Es beschreibt das Problem, eine allgemeine Lösung und Beispiele für deren
Anwendung. Ein einzelnes UI-Element, wie z. B. eine Registerkarte,
kann ebenfalls ein Design Pattern sein.

Design Patterns müssen den Gestaltungsregeln entsprechen.

## Beispiele

- Akkordeons, Registerkarten: Lösen das Problem, wie man mehr Daten anzeigt, als
  auf den Bildschirm passen.
- Wizards (Assistenten): Helfen Anfängern, komplexe Aufgaben in verständlichen
  Schritten zu erledigen.
- FAQ: Beantworten viele häufig gestellte Fragen der Nutzer

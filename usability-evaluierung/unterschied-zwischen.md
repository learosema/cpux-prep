# Unterschied zwischen Usability-Test, Usability-Inspektion und Benutzerbefragung
  
## Usability-Test vs Benutzerbefragung

Eine Benutzerbefragung:

- Fragt quantitativ nach Meinungen, Einstellungen und Zufriedenheit.
- Benutzer sagen, was sie denken oder fühlen.
- Liefert Zahlen und Stimmungen zur User Experience.
- Zeigt keine konkreten Usability-Probleme.

Ein Usability-Test:

- Beobachtet das tatsächliche Benutzerverhalten.
- Benutzer führen Aufgaben aus, statt darüber zu reden.
- Macht Usability-Probleme sichtbar, die Nutzer selbst oft nicht benennen können.

### "Meinungen" in der Usability-Evaluierung?

Usability-Evaluierung fokussiert primär das tatsächliche
Benutzerverhalten, aber sie kann ergänzend Statistiken zu Meinungen einbeziehen.

Allerdings nicht in Form von Interviews und Fokusgruppen (siehe
[Warum sind Interviews und Fokusgruppen ungeeignet?](warum-sind-interviews-und-fokusgruppen-ungeeignet.md)).

## Usability-Test vs Usability-Inspektion

Ein Usability-Test:

- Wird mit echten Benutzern durchgeführt.
- Benutzer führen Aufgaben aus.
- Es wird das tatsächliche Benutzerverhalten beobachtet.
- Zeigt reale Usability-Probleme, die Benutzer wirklich haben.

Eine Usability-Inspektion:

- Wird von Experten durchgeführt, ohne Benutzer.
- Experten prüfen das System selbst.
- Bewertung basiert auf Erfahrung, Heuristiken, Gestaltungsregeln und Styleguides.
- Findet potenzielle Usability-Probleme, keine beobachteten.

Kurz gesagt:

- Usability-Test = Benutzer benutzen das System.
- Usability-Inspektion = Experten beurteilen das System.
